## 📖 Описание проекта

### Назначение

**СВК Analytics** — это автоматизированная система анализа ежегодных сводных отчетов по мониторингу организации и осуществлению внутреннего контроля (СВК) в подведомственных организациях.

### Ключевая идея

Система реализует принцип **"положил файл — получил аналитику"**:
- Каждый год загружается новый отчет в папку `data/raw/`
- Запускается одна команда
- Автоматически генерируются расчеты, таблицы и интерактивные дашборды

### Целевая аудитория

- Специалисты по внутреннему контролю
- Аналитики государственных органов
- Руководители подведомственных организаций
- Аудиторы и контролёры

### Масштаб анализируемых данных

На основе последнего загруженного отчета (по состоянию на 22.06.2026):
- **Организаций в системе:** ~685
- **Типы организаций:** Высшее образование, Научные учреждения, Другие
- **Федеральные округа:** 8
- **Анализируемых направлений:** 4 (ФХД, Закупки, Имущество, Проектная деятельность)
- **Метрик на организацию:** 150+

## 🔬 Методы анализа

### 1. Загрузка и нормализация данных

**Модуль:** `src/svk_analytics/io.py`, `src/svk_analytics/columns.py`

#### Поддерживаемые форматы
- `.XLS` (старый формат Excel)
- `.XLSX` (новый формат Excel)
- `.HTML` (экспорт из веб-форм)

#### Автоматический поиск колонок
Система использует **регулярные выражения** для поиска колонок в отчете:
- Конфигурация: `config/columns.yml` (273 строки)
- 60+ колонок с множественными шаблонами поиска
- Устойчивость к изменениям формулировок вопросов

**Пример конфигурации:**
```yaml
columns:
  org_name:
    label: Наименование организации
    patterns:
      - "^Наименование организации$"
  
  violations_total:
    label: Выявлено нарушений
    patterns:
      - "Выявлено нарушений.*всего"
      - "количество выявленных нарушений"
```

### 2. Очистка и валидация данных

**Модуль:** `src/svk_analytics/cleaning.py`

#### Методы очистки

1. **Нормализация текста** (`clean_text`)
   - Удаление лишних пробелов
   - Приведение к нижнему регистру
   - Обработка спецсимволов

2. **Извлечение чисел** (`to_number`)
   - Парсинг числовых значений
   - Обработка разделителей (точки, запятые, пробелы)
   - Очистка от нецифровых символов

3. **Булевы флаги** (`yes_flag`)
   - Преобразование "Да"/"Нет" в 1/0
   - Обработка вариантов написания
   - Работа с пустыми значениями

### 3. Классификация организаций

**Модуль:** `src/svk_analytics/scoring.py` → `classify_org_type()`

#### Группировка типов организаций

Из 8 оригинальных типов формируются 3 категории:

1. **Высшее образование** (233 организации)
   - Университеты
   - Институты
   - Академии

2. **Научные учреждения** (404 организации)
   - НИИ
   - Исследовательские центры
   - Лаборатории

3. **Другие** (48 организаций)
   - Государственные учреждения
   - Дошкольные образовательные организации
   - Унитарные предприятия
   - СПО (среднее профессиональное образование)
   - Учреждения культуры
   - ДПО (дополнительное профессиональное образование)

### 4. Расчет покрытия направлений

**Модуль:** `src/svk_analytics/scoring.py` → `enrich_with_metrics()`

#### Анализируемые направления

Для каждого из 4 направлений:

1. **Финансово-хозяйственная деятельность (ФХД)**
   - Кассовые поступления
   - Кассовые выплаты
   - Среднесписочная численность
   - Количество операций ФХЖ

2. **Закупки**
   - Количество закупок
   - Объем закупок (руб)

3. **Имущество**
   - Здания: количество, площадь
   - Земля: участков, площадь
   - ОЦДИ: количество, стоимость
   - Операции с недвижимостью
   - Операции с ОЦДИ

4. **Проектная деятельность**
   - Количество проектов
   - Объем проектов (руб)
   - Количество контрактов
   - Объем контрактов (руб)

#### Алгоритм расчета покрытия

Для каждого направления:

1. **Определение активности**
   ```
   ACTIVE = (сумма метрик направления) > 0
   ```

2. **Проверка покрытия**
   ```
   COVERED = флаг "направление покрыто СВК" == "Да"
   ```

3. **Выявление непокрытия**
   ```
   UNCOVERED = ACTIVE AND NOT COVERED
   ```

4. **Расчет доли покрытия**
   ```
   COVERAGE_SHARE = (количество покрытых активных направлений) / 
                     (количество активных направлений)
   ```

### 5. Расчет нагрузки по направлениям

**Модуль:** `src/svk_analytics/scoring.py` → `_percentile_scores()`, `_level_from_percentile()`

#### Метод: Перцентильное ранжирование

Система использует **перцентили** для определения относительной нагрузки организации:

1. **Расчет перцентилей**
   - Для каждой метрики направления (объем закупок, количество операций ФХЖ и т.д.)
   - Вычисляется перцентильный ранг (0-100%)
   - Берется максимальный перцентиль по всем метрикам направления

2. **Уровни нагрузки** (из `config/scoring.yml`):
   ```yaml
   level_1_max_percentile: 0.75    # ≤75-й перцентиль
   level_2_max_percentile: 0.90    # 75-90-й перцентиль
   level_3_max_percentile: 0.95    # 90-95-й перцентиль
   # Level 4: >95-го перцентиля
   ```

3. **Интерпретация уровней:**
   - **Уровень 0:** Нет активности
   - **Уровень 1:** Малая нагрузка (нижние 75%)
   - **Уровень 2:** Средняя нагрузка (75-90%)
   - **Уровень 3:** Высокая нагрузка (90-95%)
   - **Уровень 4:** Очень высокая нагрузка (топ-5%)

### 6. Определение формы СВК

**Модуль:** `src/svk_analytics/scoring.py` → расчет `svk_form_level`

#### Формы СВК (иерархия)

Система выделяет 4 формы внутреннего контроля по возрастанию "силы":

1. **Уровень 1:** Уполномоченное должностное лицо
   - Один сотрудник отвечает за СВК
   - Минимальная форма

2. **Уровень 2:** Временный коллегиальный орган / комиссия
   - Группа сотрудников, собирается по необходимости
   - Нет постоянного функционала

3. **Уровень 3:** Постоянно действующий коллегиальный орган
   - Постоянная комиссия
   - Регулярные заседания

4. **Уровень 4:** Уполномоченное структурное подразделение
   - Отдельное подразделение (отдел, управление)
   - Максимальная форма

#### Алгоритм определения фактической формы

```python
svk_form_level = MAX(
    form_responsible_person * 1,
    form_temporary_body * 2,
    form_permanent_body * 3,
    form_structural_unit * 4
)
```

Где флаги принимают значение 1 (если отмечено) или 0.

### 7. Расчет требуемой формы СВК

**Модуль:** `src/svk_analytics/scoring.py` → расчет `required_form_level`

#### Базовое правило
```
REQUIRED_FORM = MAX(уровень нагрузки по всем направлениям)
```

#### Правило сложности (complexity bump)

Если у организации **3 и более направлений** с нагрузкой ≥2 уровня:
```
REQUIRED_FORM += 1
```

**Обоснование:** Множественные направления с высокой нагрузкой требуют более сложной формы координации.

**Конфигурация:**
```yaml
complexity_bump:
  enabled: true
  min_directions_count: 3      # Минимум направлений
  min_direction_level: 2       # Минимальный уровень нагрузки
  bump_by: 1                   # Повышение на 1 уровень
```

### 8. Классификация рисков

**Модуль:** `src/svk_analytics/scoring.py` → расчет `risk_group`

#### Группы риска (5 категорий)

Система автоматически относит организацию к одной из групп:

| Группа | Название | Условие | Приоритет внимания |
|--------|----------|---------|-------------------|
| **A** | Сбалансированная СВК | Направления покрыты, форма адекватна | ✅ Низкий |
| **B** | Недопокрытие активных направлений | `UNCOVERED > 0` | ⚠️ Средний |
| **C** | Слабая форма СВК | `FORM_GAP < 0` (фактическая < требуемой) | ⚠️ Средний |
| **D** | Двойной риск | `UNCOVERED > 0` И `FORM_GAP < 0` | 🔴 Высокий |
| **E** | Отчет не заполнен / данные требуют проверки | Статус отчета = "не заполнен" | ❌ Критический |

#### Алгоритм классификации (приоритет сверху вниз)

```python
IF report_status == "не заполнен":
    → Группа E
ELIF uncovered_count > 0 AND form_gap < 0:
    → Группа D (двойной риск)
ELIF uncovered_count > 0:
    → Группа B (непокрытие)
ELIF form_gap < 0:
    → Группа C (слабая форма)
ELSE:
    → Группа A (сбалансировано)
```

### 9. Детектирование противоречий

**Модуль:** `src/svk_analytics/scoring.py` → `build_contradictions()`

#### Типы логических противоречий

1. **Устранено > Выявленных нарушений**
   ```
   (violations_fixed_on_time + violations_fixed_late) > violations_total
   ```

2. **Отменено > Принятых дисциплинарных решений**
   ```
   cancelled_decisions > disciplinary_decisions
   ```

3. **Возвращено > Направленных в правоохранительные органы**
   ```
   returned_refused > materials_law_enforcement
   ```

4. **Отказано > Направленных в суд**
   ```
   refused_satisfaction > materials_court
   ```

5. **СВК не организован, но заполнены элементы СВК**
   ```
   svk_organized == "Нет" AND (accounting_policy == "Да" OR 
                                lna_approved == "Да" OR ...)
   ```

## 📊 Метрики и показатели

### Обзорные метрики (12 KPI)

| № | Метрика | Описание | Расчет |
|---|---------|----------|--------|
| 1 | **Организаций** | Общее количество организаций | `COUNT(*)` |
| 2 | **Заполненность отчетов** | % заполненных отчетов | `SUM(is_report_filled) / COUNT(*)` |
| 3 | **СВК организован** | % организаций, где СВК организован | `SUM(svk_organized == "Да") / COUNT(*)` |
| 4 | **Полный базовый комплект** | % с полным комплектом базовых элементов | `SUM(ALL(4 элемента)) / COUNT(*)` |
| 5 | **Полноценная методика рисков** | % с полноценной методикой оценки рисков | `SUM(risk_methodology == полная) / COUNT(*)` |
| 6 | **Регулярная оценка СВК** | % с регулярной оценкой эффективности | `SUM(effectiveness_review == регулярная) / COUNT(*)` |
| 7 | **Нарушения** | Общее количество выявленных нарушений | `SUM(violations_total)` |
| 8 | **Устранено в срок** | % нарушений, устраненных в срок | `SUM(violations_fixed_on_time) / SUM(violations_total)` |
| 9 | **Покрытие активных направлений** | Средняя доля покрытых активных направлений | `AVG(coverage_share_active)` |
| 10 | **Непокрытые направления** | Организаций с непокрытыми направлениями | `COUNT(uncovered_count > 0)` |
| 11 | **Слабая форма СВК** | Организаций со слабой формой | `COUNT(form_gap < 0)` |
| 12 | **Двойной риск** | Организаций с двойным риском | `COUNT(risk_group == "D.*")` |

### Метрики нарушений

| Метрика | Описание | Источник |
|---------|----------|----------|
| Выявлено нарушений | Общее количество | `violations_total` |
| Устранено в срок | В установленный срок | `violations_fixed_on_time` |
| Устранено с просрочкой | После установленного срока | `violations_fixed_late` |
| В процессе устранения | Еще не устранены | `violations_in_progress` |
| Не подлежат устранению | По объективным причинам | `violations_not_subject` |

### Типичные проблемы качества

#### Противоречия (логические несоответствия)

1. **Арифметические противоречия**
   - Устранено нарушений больше, чем выявлено
   - Отменено решений больше, чем принято

2. **Логические противоречия**
   - СВК не организован, но заполнены элементы СВК
   - Есть нарушения, но нет мер по устранению

#### Аномалии (подозрительные паттерны)

1. **Экстремальные значения**
   - Топ-1% по любой метрике
   - Могут быть легитимными (крупные организации)

2. **Паттерны ошибок**
   - Одинаковые значения в разных полях (копипаст)
   - Нули в важных полях при большой организации

3. **Несоответствия ожиданиям**
   - Ноль нарушений при большом штате
   - Ноль закупок при большом бюджете

### Автоматические проверки

Система проверяет **5 типов противоречий** и **5 типов аномалий** для каждой организации.

Результаты:
- Текстовое описание проблемы в колонках `contradictions`, `anomalies`
- Флаги качества в колонке `data_quality_flags`
- Отдельные таблицы для противоречий и аномалий
