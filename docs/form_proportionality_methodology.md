# Методология оценки соразмерности формы СВК

Документ описывает, **какие данные** используются, **какими методами** они обрабатываются и **почему** выбран именно такой подход. Примеры приведены на реальных организациях из отчёта 2025 года (685 организаций).

**Реализация:** `src/svk_analytics/scoring.py` → функция `enrich_with_metrics()`  
**Конфигурация:** `config/scoring.yml`, `config/columns.yml`  
**Связанный паспорт метрик:** `docs/metric_passport.md` (разделы 2–7)

---

## 1. Цель и результат

**Цель:** оценить, соответствует ли организационная форма системы внутреннего контроля (СВК) масштабу и сложности деятельности организации.

**Итоговые метрики соразмерности:**

```text
form_gap = svk_form_level − recommended_form_level   # отклонение от модельной рекомендации
form_vs_peer = svk_form_level − peer_form_median     # отклонение от медианы аналогов
```

| `form_gap` | Интерпретация (модельная рекомендация) |
|---:|---|
| ≥ +1 | Форма сильнее рекомендуемой |
| 0 | Форма соответствует рекомендации |
| −1 | Умеренный дефицит формы |
| −2 | Существенный дефицит формы |
| −3 / −4 | Критический дефицит формы |

**Распределение по отчёту 2025:**

| `form_gap` | Организаций |
|---:|---:|
| −4 | 28 |
| −3 | 17 |
| −2 | 73 |
| −1 | 118 |
| 0 | 185 |
| +1 | 110 |
| +2 | 142 |
| +3 | 12 |

## 2. Общая схема расчёта

```text
Исходный отчёт (Excel)
        │
        ▼
Сопоставление колонок (columns.yml)
        │
        ▼
┌───────────────────────────────────────────────────────────┐
│  Для каждого направления деятельности:                     │
│    1. Определить активность                               │
│    2. Рассчитать относительную нагрузку (перцентили)      │
│    3. Перевести в уровень 0–4                             │
└───────────────────────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────────────────────┐
│  Фактическая форма СВК (из отметок в отчёте)              │
│    → svk_form_level (0–4)                                 │
└───────────────────────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────────────────────┐
│  Расчётно рекомендуемая форма СВК                         │
│    = max(уровни нагрузки по направлениям)                 │
│    + поправка на комплексность (при необходимости)        │
│    → recommended_form_level (0–4)                         │
└───────────────────────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────────────────────┐
│  Сравнение с аналогами (peer benchmark)                  │
│    → form_vs_peer, coverage_vs_peer                       │
└───────────────────────────────────────────────────────────┘
        │
        ▼
   form_gap = svk_form_level − recommended_form_level
```

Соразмерность формы оценивается **двумя способами**: относительно расчётной рекомендации модели и относительно практики организаций с похожей нагрузкой.

## 3. Исходные данные

### 3.1. Формы СВК (фактическая структура)

Из отчёта считываются четыре взаимоисключающих по смыслу, но не по заполнению варианта:

| Поле в системе | Уровень | Форма |
|---|---:|---|
| `form_responsible_person` | 1 | Уполномоченное должностное лицо |
| `form_temporary_body` | 2 | Временный коллегиальный орган / комиссия |
| `form_permanent_body` | 3 | Постоянно действующий коллегиальный орган |
| `form_structural_unit` | 4 | Уполномоченное структурное подразделение |

Ответ «Да» / «Нет» преобразуется в флаг 1 / 0.

**Почему шкала 1–4:** каждый следующий уровень предполагает большую организационную ёмкость — от одного ответственного лица до выделенного подразделения. Это отражает возрастание «силы» формы СВК и согласуется с вариантами, предусмотренными в сводном отчёте.

### 3.2. Направления деятельности и показатели нагрузки

Четыре направления внутреннего контроля:

| Ключ | Направление | Признак покрытия | Показатели нагрузки |
|---|---|---|---|
| `fhd` | Финансово-хозяйственная деятельность | `cover_fhd` | кассовые поступления и выплаты, среднесписочная численность, количество ФХЖ |
| `procurement` | Закупки | `cover_procurement` | количество и объём закупок |
| `property` | Имущество | `cover_property` | здания, земля, ОСДИ, операции с недвижимостью и ОСДИ |
| `project` | Проектная деятельность | `cover_project` | количество и объём проектов, договоры |

**Почему четыре направления:** это виды деятельности, по которым организация в отчёте указывает и факт операций, и включение направления в контур СВК. Нагрузка по направлению определяет «вес» деятельности; покрытие — отдельный показатель (используется в группах риска, но не в `form_gap`).

---

## 4. Метод 1. Определение активности направления

**Метод:** направление считается **активным**, если сумма числовых показателей по нему больше нуля.

**Пример — Агрофизический НИИ:**

| Направление | Показатели | Сумма | Активно? |
|---|---|---:|---|
| ФХД | касса, персонал, ФХЖ | > 0 | Да |
| Закупки | 188 закупок, 19,6 млн руб. | > 0 | Да |
| Имущество | здания, земля, ОСДИ | > 0 | Да |
| Проекты | 5 договоров | > 0 | Да |

**Обоснование:** если по направлению нет операций, оно не создаёт нагрузку на СВК и не должно повышать рекомендуемую форму. Нулевая активность даёт `load_level = 0` независимо от того, отмечено ли направление как «покрытое».

---

## 5. Метод 2. Оценка нагрузки через перцентильное ранжирование

### 5.1. Алгоритм

Для каждого числового показателя направления:

1. Берутся только организации с **положительным** значением показателя.
2. Каждой организации присваивается **перцентильный ранг** (0–1) — доля организаций с меньшим или равным значением.
3. По направлению берётся **максимум** перцентилей по всем показателям → `load_score`.
4. `load_score` переводится в уровень нагрузки 0–4 по порогам.

**Пороги** (`config/scoring.yml`):

| Уровень | Условие |
|---:|---|
| 0 | Нет активности |
| 1 | Активность есть, перцентиль ≤ 0,75 |
| 2 | Перцентиль > 0,75 и ≤ 0,90 |
| 3 | Перцентиль > 0,90 и ≤ 0,95 |
| 4 | Перцентиль > 0,95 |

### 5.2. Почему перцентили, а не абсолютные пороги

- Сводный отчёт охватывает организации **разного масштаба** (от небольших учреждений до крупных вузов и НИИ). Фиксированный порог в рублях или штуках был бы несправедлив к малым организациям и нечувствителен к крупным.
- Перцентиль отвечает на вопрос: **«насколько эта организация крупнее остальных в данном отчёте?»** — без привязки к конкретным единицам измерения.
- При замене годового файла пороги **автоматически пересчитываются** под новый массив данных.

### 5.3. Почему максимум по показателям направления

По одному направлению у организации может быть, например, мало закупок по количеству, но большой объём по сумме. Максимум перцентиля фиксирует **наиболее «тяжёлую» грань** деятельности в этом направлении. Усреднение занизило бы нагрузку у организаций с выраженным пиком в одном показателе.

### 5.4. Пример расчёта нагрузки

**Агрофизический НИИ — направление «Закупки»:**

| Показатель | Значение | Перцентиль среди организаций с закупками |
|---|---:|---:|
| Количество закупок | 188 | 25,3% |
| Объём закупок | 19 599 180 руб. | (отдельный перцентиль) |

`load_score` = max(перцентилей) ≈ **0,25** → уровень **1** (активность есть, но ниже 75-го перцентиля).

**Аграрный научный центр «Донской» — направление «Имущество»:**

| Показатель | Значение | Перцентиль |
|---|---:|---:|
| Стоимость ОСДИ | 1 523 490 684 руб. | 98,7% |

`load_score` ≈ **0,99** → уровень **4** (топ-5% по массиву отчёта).

## 6. Метод 3. Определение фактической формы СВК

### 6.1. Алгоритм

```text
svk_form_level = MAX(отмеченные формы × их уровни)
```

Если ни одна из четырёх форм не отмечена → уровень **0** («Иная форма»).

### 6.2. Почему берётся максимум

Организация может отметить несколько форм одновременно (например, и временный орган, и постоянный). Для оценки соразмерности важна **наиболее развитая** из заявленных структур — она определяет реальную организационную ёмкость СВК.

### 6.3. Примеры

**Алтайский государственный технический университет** (`form_gap = +2`):

| Форма | Отмечена? | Уровень |
|---|---|---:|
| Уполномоченное должностное лицо | Нет | — |
| Временный коллегиальный орган | **Да** | 2 |
| Постоянно действующий орган | **Да** | 3 |
| Структурное подразделение | **Да** | 4 |

→ `svk_form_level = max(2, 3, 4) = **4**` (структурное подразделение).

**Агрофизический НИИ** (`form_gap = 0`):

| Форма | Отмечена? |
|---|---|
| Временный коллегиальный орган | **Да** |
| Остальные | Нет |

→ `svk_form_level = **2**`.

## 7. Метод 4. Расчёт рекомендуемой формы СВК

Рекомендуемая форма отвечает на вопрос: **какая организационная «мощность» СВК рекомендуется моделью при таком объёме и разнообразии деятельности?** Это эвристика, а не нормативное требование.

### 7.1. Базовое правило

```text
recommended_form_level = max(уровень нагрузки по всем направлениям)
```

Логика: если хотя бы одно направление имеет очень высокую нагрузку, СВК должна обладать соответствующей организационной ёмкостью.

**Пример — Агрофизический НИИ:**

| Направление | Уровень нагрузки |
|---|---:|
| ФХД | 1 |
| Закупки | 1 |
| Имущество | **2** |
| Проекты | 1 |

→ `max = **2**` → рекомендуется форма уровня 2 (временный коллегиальный орган).

### 7.2. Поправка на комплексность (complexity bump)

**Правило:** если **3 и более направлений** имеют нагрузку **≥ 2**, рекомендуемая форма **повышается на 1 уровень** (но не выше 4).

Параметры в `config/scoring.yml`:

```yaml
complexity_bump:
  enabled: true
  min_directions_count: 3   # минимум направлений
  min_direction_level: 2    # минимальный уровень нагрузки
  bump_by: 1                # на сколько повысить
```

**Почему нужна поправка:** организация с высокой нагрузкой в **одном** направлении и организация с **умеренной** нагрузкой в **трёх–четырёх** направлениях предъявляют разные требования к координации. Многонаправленность увеличивает сложность управления внутренним контролем даже при умеренных уровнях по каждому направлению в отдельности.

### 7.3. Пример с поправкой — Адыгейский государственный университет

| Направление | Уровень нагрузки | ≥ 2? |
|---|---:|---|
| ФХД | 2 | Да |
| Закупки | 2 | Да |
| Имущество | 2 | Да |
| Проекты | 2 | Да |

Расчёт:

```text
max(нагрузки)           = 2
направлений с нагрузкой ≥ 2 = 4  (≥ 3 → поправка срабатывает)
recommended_form_level     = 2 + 1 = 3
```

Фактическая форма: структурное подразделение (уровень **4**).

```text
form_gap = 4 − 3 = +1
```

Форма сильнее рекомендуемой, хотя базовая нагрузка по каждому направлению — лишь уровень 2.

### 7.4. Пример без поправки — Агрофизический НИИ

| Направление | Уровень нагрузки | ≥ 2? |
|---|---:|---|
| ФХД | 1 | Нет |
| Закупки | 1 | Нет |
| Имущество | 2 | Да |
| Проекты | 1 | Нет |

Направлений с нагрузкой ≥ 2: **1** (< 3) → поправка **не** применяется.

```text
recommended_form_level = 2
svk_form_level      = 2
form_gap            = 0   ← соразмерность
```

### 7.5. Пример с высокой пиковой нагрузкой — Аграрный научный центр «Донской»

| Направление | Уровень нагрузки | ≥ 2? |
|---|---:|---|
| ФХД | 2 | Да |
| Закупки | 2 | Да |
| Имущество | **4** | Да |
| Проекты | 0 (неактивно) | — |

```text
max(нагрузки)               = 4
направлений с нагрузкой ≥ 2 = 3  → поправка +1
recommended_form_level         = min(4 + 1, 4) = 4
```

Фактическая форма: постоянный коллегиальный орган + временный орган → `svk_form_level = **3**`.

```text
form_gap = 3 − 4 = −1   ← умеренный дефицит формы
```

Все активные направления покрыты СВК (покрытие 100%), но форма ниже рекомендуемой из-за очень высокой нагрузки по имуществу (топ-5% массива).

**Важно — расхождение с peer-оценкой:** у «Донского» `form_gap = −1`, но среди 11 аналогов с `max_load=4` и 3 активными направлениями медиана формы = **2**, а фактическая форма = **3** → `form_vs_peer = +1`. Модель рекомендует форму 4 из-за пиковой нагрузки; большинство аналогов устроило меньшее.

## 8. Метод 5. Сравнение с организациями-аналогами (peer benchmark)

### 8.1. Алгоритм

Для каждой организации формируется группа аналогов с похожей нагрузкой. Группировка с откатом при малом размере:

1. **Профиль нагрузки** — вектор уровней по 4 направлениям (например, `2-2-4-0`).
2. Если в группе < 5 организаций → **max нагрузка + число активных направлений** (например, `4_act3`).
3. Если снова < 5 → только **max нагрузка** (например, `max_4`).

По группе считаются медианы формы СВК и доли покрытия активных направлений:

```text
form_vs_peer = svk_form_level − peer_form_median
coverage_vs_peer = coverage_share_active − peer_coverage_median
```

### 8.2. Обоснование

- Отвечает на вопрос **«как делают другие с похожей нагрузкой?»**, а не «что говорит формула».
- Медиана устойчива к выбросам и отражает типичную практику в отчёте.
- Откат группировки гарантирует достаточный размер выборки для смыслового сравнения.

### 8.3. Пример — Аграрный центр «Донской»

| Показатель | Значение |
|---|---|
| Группа аналогов | `4_act3`, n = 11 |
| Медиана формы у аналогов | 2 |
| Фактическая форма | 3 |
| `form_vs_peer` | **+1** |
| Медиана покрытия у аналогов | 100% |
| `coverage_vs_peer` | **0** |

### 8.4. Пример — Академия комфорта

| Показатель | Значение |
|---|---|
| Группа аналогов | `4_act4`, n = 172 |
| Медиана формы | 3 |
| Фактическая форма | 2 |
| `form_vs_peer` | **−1** |
| Медиана покрытия | 75% |
| Фактическое покрытие | 50% |
| `coverage_vs_peer` | **−0,25** |

## 9. Метод 6. Отклонение от рекомендуемой формы

```text
form_gap = svk_form_level − recommended_form_level
```

### 9.2. Сводная таблица примеров

| Организация | Факт. форма | Рек. форма | `form_gap` | Суть |
|---|---:|---:|---:|---|
| Агрофизический НИИ | 2 | 2 | **0** | Нагрузка умеренная, форма соответствует |
| Аграрный центр «Донской» | 3 | 4 | **−1** | Высокая нагрузка по имуществу, форма на уровень ниже |
| Академия комфорта | 2 | 4 | **−2** | Пиковая нагрузка по имуществу + слабая форма |
| Адыгейский госуниверситет | 4 | 3 | **+1** | Структурное подразделение при умеренной многонаправленности |
| Алтайский политех | 4 | 2 | **+2** | Форма избыточна относительно нагрузки |

### 9.3. Пример дефицита формы и двойного риска — Академия комфорта

| Показатель | Значение |
|---|---|
| Фактическая форма | 2 (временный орган) |
| Макс. нагрузка | 4 (имущество, топ-5%) |
| Направлений с нагрузкой ≥ 2 | 1 → поправка не срабатывает |
| Рекомендуемая форма | **4** |
| `form_gap` | **−2** |
| Покрытие активных направлений | 50% (имущество и проекты не покрыты) |
| Группа риска | D. Двойной риск |

Здесь видно различие двух осей анализа:

- **Соразмерность формы** (`form_gap`) — соответствует ли структура СВК масштабу деятельности.
- **Покрытие направлений** — включены ли активные направления в контур СВК.

Организация может иметь формально «сильную» структуру, но не покрывать направления (и наоборот). Академия комфорта — случай, когда **оба** показателя в зоне риска.

---

## 10. Связь с группами риска

Группы риска используют `form_gap` наряду с покрытием направлений. Приоритет проверки условий — сверху вниз:

| Группа | Условие |
|---|---|
| E | Отчёт не заполнен |
| D | Есть непокрытые направления **и** `form_gap < 0` |
| B | Есть непокрытые направления |
| C | `form_gap < 0` (форма слабее рекомендуемой) |
| A | Покрытие полное, форма соразмерна |

**Важно:** `form_gap = 0` не гарантирует группу A. Например, Агрофизический НИИ имеет `form_gap = 0`, но относится к группе **B**, потому что проектная деятельность активна, но не покрыта СВК.

| API `/api/form-gap` | Peer- и рекомендуемые метрики для фронтенда |
| Карточка «Слабая форма СВК» | Число организаций с `form_gap < 0` |
| `top_risk_organizations()` | Приоритизация: дефицит формы учитывается с весом 3 в `risk_sort` |
| `by_dimension_summary()` | Средний `form_gap` и число организаций со слабой формой по округам |

---

## 12. Ограничения и допущения

1. **Относительность нагрузки.** Уровни 1–4 зависят от состава организаций в конкретном годовом отчёте. Сравнение `load_level` между разными годами возможно только при сопоставимом массиве.

2. **Эвристическая модель.** Пороги перцентилей (75 / 90 / 95) и правило complexity bump (3 направления, уровень ≥ 2) заданы в конфигурации и могут быть скорректированы без изменения кода.

3. **Самоотчётность.** Фактическая форма определяется по отметкам организации в анкете, без независимой верификации структуры.

4. **Направления с нулевой активностью** не повышают рекомендуемую форму, даже если формально отмечены как покрытые.

5. **Множественные формы.** В расчёте `form_gap` учитывается только максимальный уровень; сочетание отмеченных форм анализируется отдельно (вкладка «Зрелость СВК»).

6. **Peer-сравнение** отражает практику внутри годового отчёта; «большинство аналогов» не означает «нормативно правильно».

## 13. Настройка параметров

Все пороги вынесены в `config/scoring.yml`:

```yaml
load_level_thresholds:
  level_1_max_percentile: 0.75
  level_2_max_percentile: 0.90
  level_3_max_percentile: 0.95

complexity_bump:
  enabled: true
  min_directions_count: 3
  min_direction_level: 2
  bump_by: 1

peer_benchmark:
  min_group_size: 5
```

Изменение этих значений пересчитывает `recommended_form_level`, `form_gap`, `form_vs_peer` и `coverage_vs_peer` для всего массива при следующей загрузке отчёта.
