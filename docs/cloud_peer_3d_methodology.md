# Методология сравнения с аналогами в 3D-облаке

Документ описывает блок сравнения формы СВК с медианой ближайших организаций
в пространстве трёх показателей ФХД.

**Реализация:** `src/svk_analytics/summaries.py` → `cloud_peer_benchmark()`, `cloud_peer_benchmark_table()`  
**Интерфейс:** `app.py` → вкладка «Распределение 3D»  
**Связанные документы:** `docs/form_proportionality_methodology.md`, `docs/anomaly_3axis_methodology.md`, `docs/scale_axis_profile_methodology.md`

---

## 1. Цель

Ответить на вопрос: *насколько форма СВК соответствует типичной у организаций
близкого масштаба* (поступления, численность, ФХЖ), а не только у организаций
с похожим профилем нагрузки (`form_vs_peer`).

## 2. Три оси

| Ось | Колонка |
|-----|---------|
| Сумма кассовых поступлений | `cash_receipts` |
| Среднесписочная численность | `staff_avg` |
| Количество фактов ФХЖ | `fkhz_count` |

Координаты: `ln(1 + x)` по каждой оси.

## 3. Группа аналогов

Для организации *i* находятся **k ближайших соседей** по евклидову расстоянию
в 3D log-пространстве (без самой организации), где  
`k = peer_benchmark.min_group_size` (по умолчанию 5).

## 4. Метрики

| Метрика | Формула | Смысл |
|---------|---------|-------|
| `peer_3d_form_median` | `round(median(svk_form_level соседей))` | Типичная форма у соседей |
| `form_vs_peer_3d` | `svk_form_level − peer_3d_form_median` | Отклонение формы |
| `profile_skew_3d` | `max_j |ln(1+x_ij) − median_j(соседи)|` | Перекос профиля масштаба |
| `profile_skew_3d_axis` | ось с макс. перекосом | `cash_receipts` / `staff_avg` / `fkhz_count` |
| `peer_3d_confidence` | `size_factor × (1 − rank(mean_distance))` | Надёжность сравнения (0..1) |

Порог приоритета в UI: `form_vs_peer_3d ≤ −1` и `peer_3d_confidence ≥ 0.5`.

## 5. Отличие от `form_vs_peer`

| | `form_vs_peer` | `form_vs_peer_3d` |
|---|----------------|-------------------|
| Группировка | Профиль нагрузки по 4 направлениям | k ближайших в 3D-облаке |
| Вопрос | Как у организаций с такой же нагрузкой? | Как у организаций такого же масштаба? |

## 6. Ограничения

1. Требуется не менее `k + 1` заполненных отчётов в выборке.
2. Сравнение относительно текущего годового отчёта.
3. Не заменяет модельную рекомендацию (`form_gap`) и peer по нагрузке (`form_vs_peer`).
