# 🚀 Установка СВК Analytics (Streamlit)

## 📱 Онлайн версия (рекомендуется)

После деплоя на Streamlit Cloud, приложение будет доступно по адресу:
```
https://ваш-username-svk-analytics-streamlit.streamlit.app
```

### Использование онлайн версии:

1. Откройте ссылку в браузере
2. В боковой панели нажмите "Browse files"
3. Загрузите файл отчета (XLS/XLSX/HTML)
4. Выберите год анализа
5. Изучайте интерактивный дашборд с 7 вкладками

## 💻 Локальная установка

### Требования:
- Python 3.10 или выше
- Git

### Установка на Windows:

```powershell
# 1. Клонировать репозиторий
git clone https://github.com/ваш-username/svk-analytics-streamlit.git
cd svk-analytics-streamlit

# 2. Создать виртуальное окружение
python -m venv .venv

# 3. Активировать окружение
.venv\Scripts\Activate.ps1

# 4. Установить зависимости
pip install -r requirements.txt

# 5. (Опционально) Положить данные
# Скопируйте файл отчета в папку data/raw/

# 6. Запустить приложение
streamlit run app.py
```

Приложение автоматически откроется в браузере: http://localhost:8501

### Установка на macOS/Linux:


Приложение автоматически откроется в браузере: http://localhost:8501

## 🐳 Docker установка (для продвинутых пользователей)

```bash

# Запуск контейнера

  -v $(pwd)/data:/app/data \
  -v $(pwd)/outputs:/app/outputs \
  svk-analytics-streamlit

# Открыть http://localhost:8501
```

## 📊 Структура дашборда

### Главная страница - Ключевые показатели (12 KPI):
1. Организаций в системе
2. Заполненность отчетов
3. СВК организован
4. Полный базовый комплект СВК
5. Полноценная методика рисков
6. Регулярная оценка СВК
7. Выявленные нарушения
8. Устранено в срок
9. Покрытие активных направлений
10. Непокрытые направления
11. Слабая форма СВК
12. Двойной риск

### Вкладка 1: Общая статистика
- Обзорная таблица метрик
- Нарушения и их устранение (график + таблица)
- Управленческие меры (планы, решения)
- Меры юридической ответственности (дисциплинарная, правоохранительные органы, суд)
- Нормированные показатели активности

### Вкладка 2: Зрелость СВК
- Базовые элементы СВК (4 элемента)
- Фактическая форма СВК по максимальному уровню
- Отмеченные формы СВК (4 формы)
- Рискоориентированный подход
- Оценка эффективности СВК
- Совершенствование СВК

### Вкладка 3: Направления
- Покрытие активных направлений внутренним контролем
- 4 направления: ФХД, Закупки, Имущество, Проектная деятельность
- Визуализация покрытых и непокрытых организаций

### Вкладка 4: Соразмерность формы
- Разрыв между расчетно необходимой и фактической формой СВК
- Распределение разрыва (histogram)
- Scatter plot: фактическая vs требуемая форма
- Разрез по федеральным округам

### Вкладка 5: Организации в зоне риска
- Топ-100 организаций для управленческого внимания
- Группы риска: A, B, C, D, E
- Экспорт в CSV

### Вкладка 6: Качество данных
- Организации с противоречиями (логические несоответствия)
- Организации с аномалиями (подозрительные паттерны)
- Статус отчетности
- Экспорт проблемных организаций

### Вкладка 7: Настройки колонок
- Статус резолюции колонок
- Найденные и ненайденные колонки
- Помощь в настройке config/columns.yml

## ⚙️ Настройка

### Изменение года анализа:
В боковой панели используйте поле "Год анализа" (по умолчанию: 2025)

### Изменение правил расчета:
Отредактируйте файлы конфигурации:
- `config/columns.yml` — карта колонок отчета (если изменилась структура)
- `config/scoring.yml` — правила расчета нагрузки, пороги перцентилей, формы СВК

### Изменение лимита загрузки файла:
В файле `.streamlit/config.toml`:
```toml
[server]
maxUploadSize = 500  # Размер в MB
```

## 🔧 Возможные проблемы

### Проблема 1: "Error: Cannot find column 'название_колонки'"

**Причина:** Изменилась структура отчета, колонка не найдена

**Решение:**
1. Откройте вкладку "Настройки колонок"
2. Найдите ненайденные колонки
3. Отредактируйте `config/columns.yml`, добавив новые паттерны поиска

### Проблема 2: Файл не загружается

**Причина:** Превышен лимит размера (500 MB по умолчанию)

**Решение:**
- Увеличьте `maxUploadSize` в `.streamlit/config.toml`
- Или сожмите файл (удалите лишние листы в Excel)

### Проблема 3: Ошибка при установке зависимостей

**Причина:** Конфликт версий Python или библиотек

**Решение:**
```bash
# Обновите pip
python -m pip install --upgrade pip

# Переустановите зависимости
pip install --no-cache-dir -r requirements.txt
```

### Проблема 4: Streamlit Cloud показывает ошибку памяти

**Причина:** Превышен лимит памяти (1 GB на бесплатном тарифе)

**Решение:**
- Оптимизируйте загрузку данных
- Используйте `@st.cache_data` (уже используется в `app.py`)
- Рассмотрите платный тариф Streamlit Cloud

## 🚀 Деплой на Streamlit Cloud

### Шаг 1: Подготовка репозитория

```bash
git init
git add .
git commit -m "Initial commit - SVK Analytics Streamlit app"
```

### Шаг 2: Загрузка на GitHub

Создайте репозиторий на GitHub и загрузите код:
```bash
git remote add origin https://github.com/ваш-username/svk-analytics-streamlit.git
git branch -M main
git push -u origin main
```

### Шаг 3: Деплой

1. Зайдите на https://streamlit.io/cloud
2. Войдите через GitHub аккаунт
3. Нажмите "New app"
4. Выберите репозиторий: `svk-analytics-streamlit`
5. Выберите ветку: `main`
6. Укажите main file: `app.py`
7. (Опционально) Advanced settings → Python version: `3.11`
8. Нажмите "Deploy!"

### Шаг 4: Ожидание (3-5 минут)

Streamlit Cloud автоматически:
- Установит зависимости из `requirements.txt`
- Соберет приложение
- Предоставит публичный URL

### Шаг 5: Готово!

Получите URL типа: `https://ваш-username-svk-analytics-streamlit.streamlit.app`
