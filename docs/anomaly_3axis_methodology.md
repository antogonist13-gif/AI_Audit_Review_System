# Методология анализа аномалий и соразмерности по трём осям

Документ описывает блок поиска аномалий и отклонений в организациях по трём
показателям финансово-хозяйственной деятельности с учётом формы СВК.

**Реализация:** `src/svk_analytics/summaries.py` → `proportionality_anomalies()`,
`proportionality_anomalies_table()`
**Интерфейс:** `app.py` → вкладка «Аномалии (3D)»
**Связанные документы:** `docs/metric_passport.md`, `docs/form_proportionality_methodology.md`

> Блок аддитивный: он не изменяет исходный набор данных и не влияет на расчёт
> `form_gap`, `form_vs_peer`, групп риска и остальных существующих метрик.

---

## 1. Цель

Найти организации, которые **нетипичны** или **непропорциональны** по сочетанию
трёх объёмных показателей, и сопоставить масштаб их деятельности с формой СВК
(видом организации внутреннего контроля).

## 2. Три оси

| Ось | Колонка |
|---|---|
| Сумма кассовых поступлений | `cash_receipts` |
| Среднесписочная численность | `staff_avg` |
| Количество фактов ФХЖ | `fkhz_count` |

Дополнительная (категориальная) ось — форма СВК: `svk_form_level` (0–4),
`svk_form_name`.

## 3. Предобработка

- Анализируются только заполненные отчёты (`is_report_filled`).
- Показатели сильно скошены, поэтому используются:
  - **логарифмические координаты** `ln(1 + x)` — для расстояний и графика;
  - **перцентильные ранги** (среди положительных значений) — для оценки масштаба,
    согласованно с методологией нагрузки в `scoring.py`.
- Организации без активности по всем трём осям исключаются из оценки
  многомерного выброса.

## 4. Методы

### 4.1. Совокупный масштаб и его уровень

```text
scale_score = среднее перцентильных рангов трёх осей           (0..1)
scale_level = уровень 0..4 по рангу scale_score
              (пороги P1=0.75, P2=0.90, P3=0.95 из config/scoring.yml)
```

### 4.2. Многомерный выброс (расстояние Махаланобиса)

По лог-координатам трёх осей считается квадрат расстояния Махаланобиса от центра
облака (с учётом корреляции осей, только numpy). Организация помечается выбросом,
если расстояние превышает квантиль χ² с 3 степенями свободы на уровне 0.99
(≈ 11.345).

```text
scale_outlier = (mahalanobis² > 11.345)
```

### 4.3. Соразмерность (удельные отношения)

Для каждого отношения берётся логарифм и считается **робастный z-показатель**
(медиана и MAD). Порог отклонения — |z| > 3.5.

| Отношение | Формула |
|---|---|
| Поступления на 1 сотрудника | `cash_receipts / staff_avg` |
| Факты ФХЖ на 1 сотрудника | `fkhz_count / staff_avg` |
| Поступления на 1 факт ФХЖ | `cash_receipts / fkhz_count` |

### 4.4. Рассогласование «масштаб ↔ форма»

```text
scale_vs_form = scale_level − svk_form_level
```

Текстовые признаки:
- `scale_level ≥ 3` и `svk_form_level ≤ 1` → «Крупный масштаб при слабой форме СВК»;
- `scale_level ≤ 1` и `svk_form_level ≥ 4` → «Развитая форма СВК при малом масштабе».

### 4.5. Итоговый балл

```text
anomaly_score = mahalanobis + max(|z| по отношениям) + |scale_vs_form|
```

Балл используется для ранжирования и для размера маркеров на 3D-графике.
Организация считается аномальной (`is_anomaly`), если есть хотя бы одна
текстовая причина (`anomaly_reasons`).

## 5. Интерпретация

- Аномалия — это **повод для проверки**, а не вывод о нарушении. Причины могут
  быть содержательными (специфика деятельности) или связанными с качеством данных.
- Особое внимание — организациям с крупным масштабом и слабой формой СВК
  (точки ниже диагонали на графике «масштаб × форма»).

## 6. Ограничения

1. Относительность: пороги и перцентили зависят от состава организаций в годовом отчёте.
2. Удельные отношения не считаются при нулевом знаменателе.
3. Многомерный выброс требует достаточного числа заполненных отчётов в выборке.
