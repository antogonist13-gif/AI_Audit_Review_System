# Методология профиля масштаба

Документ описывает блок «Профиль масштаба: концентрация и форма СВК» —
распределение организаций вдоль оси совокупного масштаба и связь с формой СВК.

**Реализация:** `src/svk_analytics/summaries.py` → `scale_axis_profile()`  
**Интерфейс:** `app.py` → вкладка «Распределение 3D»  
**Экспорт:** `src/svk_analytics/exports.py` → лист `scale_profile_bins`  
**Связанные документы:** `docs/cloud_peer_3d_methodology.md`, `docs/anomaly_3axis_methodology.md`

> Блок аддитивный: не изменяет исходный набор данных и не влияет на расчёт
> `form_gap`, `form_vs_peer`, `cloud_peer_benchmark` и остальных метрик.

---

## 1. Цель

Ответить на управленческий вопрос: *в каком фактическом масштабе (рубли / люди /
ФХЖ) концентрируется больше всего организаций, какова там медиана формы СВК и как
эта медиана меняется по мере роста и уменьшения масштаба?*

## 2. Данные и три оси

| Ось | Колонка |
|-----|---------|
| Сумма кассовых поступлений | `cash_receipts` |
| Среднесписочная численность | `staff_avg` |
| Количество фактов ФХЖ | `fkhz_count` |

Используются строки с заполненным отчётом (`is_report_filled`).  
Координаты: `ln(1 + x)` по каждой оси.

### Активные организации

Для расчёта PC1, `signed_scale` и бинов участвуют только **активные** организации —
у которых все три оси строго положительны (`cash_receipts > 0`, `staff_avg > 0`,
`fkhz_count > 0`).

Организации с нулём хотя бы по одной оси (в т.ч. `cash_receipts = 0` при ненулевых
staff/fkhz) **не** попадают в PC1 и бинирование: им присваивается `signed_scale = NaN`,
`active = False`. Они сохраняются в таблице `orgs` для справки, но не искажают ось
масштаба (нулевые поступления в `ln(1+0)=0` тянут позицию влево без информации
о реальном масштабе).

Минимум **20** активных организаций; иначе возвращаются пустые таблицы.

## 3. Центр облака

Центр в log-пространстве (только по активным организациям):

\[
\mu_j = \mathrm{mean}\bigl(\ln(1 + x_j)\bigr), \quad j \in \{\text{cash}, \text{staff}, \text{fkhz}\}
\]

Типичный масштаб в фактических единицах:

\[
\text{typical}_j = e^{\mu_j} - 1 \approx \exp(\mu_j) - 1
\]

На типичной выборке (~652 активных организации из ~673 заполненных) центр облака
соответствует порядка 432 млн руб. / 303 чел. / 753 ед. ФХЖ.

## 4. Ось масштаба (PC1)

Расчёт только по активным организациям:

1. Стандартизация log-координат по активной выборке:
   \[
   z_{ij} = \frac{\ln(1+x_{ij}) - \mathrm{mean}_j}{\mathrm{std}_j}
   \]
2. Ковариационная матрица \(3 \times 3\) по столбцам \(z\).
3. Первая главная компонента \(g\) — собственный вектор, соответствующий
   наибольшему собственному значению (`numpy.linalg.eigh`).
4. **Фиксация знака:** если \(\sum_j g_j < 0\), вектор умножается на \(-1\)
   (все нагрузки положительны — «больше» = положительное направление).
5. Знаковая позиция организации:
   \[
   \text{signed\_scale}_i = z_i \cdot g
   \]
   Отрицательная — мельче типичного, 0 — центр (горб), положительная — крупнее.

На типичной выборке PC1 объясняет ~64% дисперсии; нагрузки по трём осям
приблизительно равны.

## 5. Бинирование и сводка

Разбиение `signed_scale` — равноширокие бины `pd.cut`. **Число бинов** настраивается
в UI блока «Профиль масштаба» (слайдер 6–24, по умолчанию 12); для Excel-экспорта
дефолт задаётся в `config/scoring.yml` → `scale_profile.n_bins`.

Два представления по одному разбиению:

| Таблица | Назначение |
|---------|------------|
| `bins_count` | Гистограмма числа организаций (`form_median` округлена до целого) |
| `bins_share` | 100%-stacked доли форм (те же бины, `form_median` — float) |

Для каждого бина:

- `n` — число организаций;
- `signed_scale_mid` — середина интервала бина;
- `signed_scale_left`, `signed_scale_right` — границы бина по оси масштаба;
- `cash_min`, `cash_max`, `staff_min`, `staff_max`, `fkhz_min`, `fkhz_max` — min/max
  фактических значений трёх осей среди организаций бина;
- `bin_idx` — порядковый номер бина (0…n−1 после сортировки по `signed_scale_mid`);
- `form_median` — медиана `svk_form_level` (в `bins_count` округляется до целого:
  `int(round(median))` для дискретной цветовой шкалы 0–4; в `bins_share` — float);
- `form_q25`, `form_q75` — квартили `svk_form_level`;
- `form_share_0` … `form_share_4` — доли форм 0–4;
- `typical_cash`, `typical_staff`, `typical_fkhz` — `exp(mean(ln(1+x))) - 1`
  по центроиду log-координат бина;
- `low_n` — флаг, если `n < 10`.

Таблица `orgs` содержит все заполненные организации с колонками `signed_scale`,
`active` (bool).

Таблица `bin_orgs` — по одной строке на каждую активную организацию с привязкой к бину
(`bin_idx`, границы бина, `org_name`, три оси ФХД, форма СВК, `signed_scale`).

## 6. Интерпретация графиков

### Фильтр выбросов профиля

Перед расчётом профиля можно включить двустороннее отсечение по перцентилям
(отдельно по каждой из трёх осей):

- **Нижний порог** (по умолчанию p0.5%) — организации с аномально малыми значениями;
- **Верхний порог** (по умолчанию p99.5%) — организации с аномально большими значениями.

Организация отсекается, если хотя бы по одной оси значение ниже нижнего или выше
верхнего порога. Отсечённые выводятся в таблицу с колонкой `profile_trim_reason`
и не участвуют в PC1 и бинировании. Фильтр не влияет на сырой 3D-scatter и блок
«Сравнение с аналогами в облаке».

### Число бинов

Слайдер «Число бинов по оси масштаба» (6–24) управляет шириной интервалов `pd.cut`:
больше значение — уже бины и детальнее график; меньше — шире бины и более обобщённый вид.
Оба графика профиля пересчитываются с выбранным числом бинов.

### График 1 — «Распределение организаций по масштабу»

- Столбцы: число организаций в бине (`n`), бины `bins_count`.
- Цвет: округлённая медиана `svk_form_level` в бине (палитра 0→4, светлый→тёмный).
- Подпись на столбце: медиана формы и `n`.
- Бины с `low_n`: пониженная непрозрачность, пометка «n мало» в hover.
- Hover столбца: медиана формы, `n`, диапазон `signed_scale` и min/max поступлений,
  численности и ФХЖ в бине.
- Вертикальная линия на бине с максимальным `n` — «Максимум концентрации».
- Карточки «типичных» показателей над графиком — `typical_*` того же пикового бина.

### График 2 — «Структура форм СВК по уровням масштаба»

- 100%-stacked bar: доли форм 0–4 по тем же бинам, что и график 1 (`bins_count` / `bins_share`).
- Та же цветовая палитра и подписи оси X, что в графике 1.
- Вертикальная линия «Максимум концентрации» — на том же бине, что и на графике 1.
- В hover: доля (%) и число организаций по форме в бине.

### Состав бинов

Под графиками — две таблицы:

1. **Сводка по бинам** — для каждого столбца гистограммы: номер бина, диапазон
   `signed_scale`, min/max поступлений (млн руб), численности (чел), ФХЖ (ед.),
   число организаций и медиана формы СВК.
2. **Детализация** — список организаций выбранного бина (selectbox), с возможностью
   скачать CSV.

### Подписи оси X

Двухстрочные (HTML `<br>`):

1. σ-позиция бина относительно центра (`signed_scale_mid / std(signed_scale)`).
2. Типичный масштаб бина в млн руб. / чел. / ед. ФХЖ.

## 7. Ограничения

1. Требуется не менее 20 активных организаций (все три оси > 0) и наличие колонок осей.
2. Расчёт по текущей выборке (фильтры дашборда); не зависит от скрытия
   экстремумов на 3D-графике.
3. Неактивные организации не участвуют в PC1, бинах и карточках «типичного масштаба».
4. Не заменяет peer-сравнение (`cloud_peer_benchmark`) и не выявляет
   непропорциональность осей (`proportionality_anomalies`).
