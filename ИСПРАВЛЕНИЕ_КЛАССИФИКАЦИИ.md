# Исправление классификации организаций

**Дата:** 2026-06-21  
**Статус:** ✅ Исправлено

---

## Проблема

Классификация организаций работала на основе анализа **названий** организаций (поиск слов "университет", "институт", "научн" и т.д.), что было неточно.

## Решение

Классификация теперь использует **реальную колонку** из исходной таблицы:  
**"Тип организации.1"** → импортируется как `org_type`

---

## Изменения в коде

### 1. config/columns.yml
Изменен порядок паттернов для импорта детальной колонки:
```yaml
org_type:
  label: Тип организации
  patterns:
    - "^Тип организации\\.1$"    # <-- теперь первый (детальные типы)
    - "^Тип организации$"         # <-- резервный
```

### 2. src/svk_analytics/scoring.py
Функция `classify_org_type()` изменена:
- **Было:** анализ названия организации (`org_name`)
- **Стало:** использование значения из колонки (`org_type`)

Группировка 8 исходных типов в 3 категории:
```python
def classify_org_type(org_type_value: str) -> str:
    if org_type_str == "Высшее образование":
        return "Высшее образование"
    if org_type_str == "Научные учреждения":
        return "Научные учреждения"
    return "Другие"  # все остальные типы
```
