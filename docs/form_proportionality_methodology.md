# Методология оценки соразмерности формы СВК

Документ описывает **все формулы**, **порядок расчёта** и **обоснование** методов раздела «Соразмерность формы». Примеры — на отчёте 2025 года (685 организаций).

**Реализация:** `src/svk_analytics/scoring.py` → `enrich_with_metrics()`, `add_peer_benchmarks()`  
**Конфигурация:** `config/scoring.yml`, `config/columns.yml`  
**Связанный паспорт метрик:** `docs/metric_passport.md`

---

## 1. Цель

Оценить, насколько организационная форма системы внутреннего контроля (СВК) **соразмерна** масштабу и сложности деятельности организации.

Используются **два независимых взгляда**:

| Взгляд | Метрика | Вопрос |
|---|---|---|
| Модельная рекомендация | `form_gap` | Соответствует ли форма расчётной рекомендации при данной нагрузке? |
| Сравнение с аналогами | `form_vs_peer` | Отличается ли форма от типичной у организаций с похожей нагрузкой? |

Обе метрики строятся на общем фундаменте — оценке **нагрузки по направлениям** и определении **фактической формы СВК**.

---

## 2. Сводка формул

### 2.1. Входные величины

| Обозначение | Смысл | Источник |
|---|---|---|
| `Fᵢ` | Флаг формы *i* (0 или 1) | Отчёт: чекбоксы форм СВК |
| `Lᵢ` | Уровень формы *i* (1–4) | `config/scoring.yml` → `forms` |
| `xⱼ` | Числовой показатель *j* по направлению | Отчёт: операционные данные |
| `Cₖ` | Покрытие направления *k* (0 или 1) | Отчёт: включение направления в СВК |

### 2.2. Нагрузка и активность

| Метрика | Формула |
|---|---|
| Активность направления *k* | `activeₖ = 1`, если `Σ xⱼ > 0` по показателям направления, иначе `0` |
| Перцентильный ранг показателя | `pⱼ = rank(xⱼ) / N₊` среди организаций с `xⱼ > 0` (метод `average`) |
| Оценка нагрузки направления | `load_scoreₖ = max(pⱼ)` по всем показателям направления |
| Уровень нагрузки | `load_levelₖ = f(load_scoreₖ)` — см. §4.3 |
| Макс. нагрузка | `max_direction_load_level = max(load_levelₖ)` |
| Число активных направлений | `active_directions_count = Σ activeₖ` |
| Профиль нагрузки | `load_profile = "L_fhd-L_proc-L_prop-L_proj"` (уровни через дефис) |

### 2.3. Покрытие (контекст для групп риска)

| Метрика | Формула |
|---|---|
| Покрыто активных направлений | `covered_active = Σ (activeₖ × Cₖ)` |
| Доля покрытия | `coverage_share_active = covered_active / active_directions_count` (если активных > 0) |
| Непокрытая нагрузка по направлению | `uncovered_loadₖ = (activeₖ × (1 − Cₖ)) × load_levelₖ` |
| Сумма непокрытой нагрузки | `uncovered_load_sum = Σ uncovered_loadₖ` |

### 2.4. Форма СВК

| Метрика | Формула |
|---|---|
| Фактическая форма | `svk_form_level = max(Fᵢ × Lᵢ)` по отмеченным формам; если ни одна не отмечена → `0` |
| Рекомендуемая форма (база) | `base = max_direction_load_level` |
| Поправка на комплексность | `bump = 1`, если `count(load_levelₖ ≥ 2) ≥ 3`, иначе `0` |
| Рекомендуемая форма (итог) | `recommended_form_level = clip(base + bump, 0, 4)` |
| Отклонение от рекомендации | `form_gap = svk_form_level − recommended_form_level` |

### 2.5. Сравнение с аналогами (peer)

| Метрика | Формула |
|---|---|
| Медиана формы в группе | `peer_form_median = round(median(svk_form_level))` по группе аналогов |
| Отклонение от аналогов | `form_vs_peer = svk_form_level − peer_form_median` |

Группа аналогов выбирается по правилу отката (§8). Все значения `form_vs_peer` и `peer_form_median` — **целые уровни** 0–4.

## 3. Общая схема расчёта

```text
Исходный отчёт (Excel)
        │
        ▼
Сопоставление колонок (columns.yml)
        │
        ▼
┌───────────────────────────────────────────────────────────┐
│  По каждому направлению k ∈ {fhd, procurement,           │
│  property, project}:                                      │
│    activeₖ, load_scoreₖ, load_levelₖ, Cₖ                 │
└───────────────────────────────────────────────────────────┘
        │
        ▼
┌───────────────────────────────────────────────────────────┐
│  Агрегаты: max_direction_load_level, coverage_share,     │
│  load_profile, active_directions_count                    │
└───────────────────────────────────────────────────────────┘
        │
        ├──────────────────────────────┐
        ▼                              ▼
┌─────────────────────┐    ┌─────────────────────────────┐

```

## 4. Метод 1. Активность направления

### Формула

Для направления *k* с набором показателей `metricsₖ`:

```text
activeₖ = 1,  если  Σₘ∈metricsₖ  xₘ > 0
activeₖ = 0,  иначе
```

### Обоснование

Направление без фактических операций не создаёт нагрузку на СВК. Нулевая активность даёт `load_levelₖ = 0` **независимо** от того, отмечено ли направление в анкете как «покрытое». Иначе формальное покрытие «пустого» направления искажало бы оценку.

### Показатели по направлениям (`config/scoring.yml`)

| Направление | Ключ | Показатели нагрузки |
|---|---|---|
| ФХД | `fhd` | `cash_receipts`, `cash_payments`, `staff_avg`, `fkhz_count` |
| Закупки | `procurement` | `procurement_count`, `procurement_amount` |
| Имущество | `property` | `buildings_count`, `buildings_area`, `land_count`, `land_area`, `ocdi_count`, `ocdi_value`, `real_estate_ops_count`, `real_estate_ops_area`, `ocdi_ops_count`, `ocdi_ops_value` |
| Проекты | `project` | `project_count`, `project_amount`, `contracts_count`, `contracts_amount` |

### Пример — Агрофизический НИИ

Все четыре направления активны (суммы показателей > 0).

## 5. Метод 2. Оценка нагрузки (перцентильное ранжирование)

### 5.1. Перцентильный ранг

Для показателя `x` внутри годового массива организаций:

```text
p(xᵢ) = rank(xᵢ; {x : x > 0}) / N₊
```

где:
- ранжируются только организации с **положительным** значением;
- `rank` — средний ранг (`method="average"`);
- `N₊` — число организаций с `x > 0`;
- если `xᵢ ≤ 0`, то `p(xᵢ) = 0`.

### 5.2. Оценка и уровень нагрузки направления

```text
load_scoreₖ = max{ p(xₘ) : m ∈ metricsₖ }

load_levelₖ = 0,                          если activeₖ = 0
load_levelₖ = 1,                          если 0 < load_scoreₖ ≤ P₁
load_levelₖ = 2,                          если P₁ < load_scoreₖ ≤ P₂
load_levelₖ = 3,                          если P₂ < load_scoreₖ ≤ P₃
load_levelₖ = 4,                          если load_scoreₖ > P₃
```

Пороги по умолчанию: `P₁ = 0,75`, `P₂ = 0,90`, `P₃ = 0,95`.

### 5.3. Максимальная нагрузка

```text
max_direction_load_level = max{ load_levelₖ : k ∈ directions }
```

### Обоснование

| Решение | Почему |
|---|---|
| Перцентили вместо абсолютных порогов | Организации в отчёте разного масштаба; сравнение «внутри массива» адаптируется к новому файлу каждый год |
| Ранжирование только положительных значений | Нули не участвуют в перцентиле — иначе большинство «малых» организаций получало бы одинаковый низкий ранг |
| Максимум по показателям направления | Фиксирует наиболее «тяжёлую» грань деятельности; усреднение занизило бы нагрузку при пике в одном показателе (например, мало закупок по количеству, но большой объём по сумме) |
| Четыре уровня + ноль | Достаточная детализация для управленческого мониторинга без избыточной гранулярности |

### Пример — Агрофизический НИИ, закупки

| Показатель | Значение | Перцентиль |
|---|---:|---:|
| `procurement_count` | 188 | 25,3% |

`load_score = 0,25` → `load_level = 1`.

### Пример — Аграрный центр «Донской», имущество

| Показатель | Значение | Перцентиль |
|---|---:|---:|
| `ocdi_value` | 1 523 490 684 руб. | 98,7% |

`load_score ≈ 0,99` → `load_level = 4` (топ-5% массива).

## 6. Метод 3. Фактическая форма СВК

### Формула

Для форм `i ∈ {responsible_person, temporary_body, permanent_body, structural_unit}` с уровнями `Lᵢ ∈ {1,2,3,4}`:

```text
svk_form_level = max{ Fᵢ × Lᵢ }
```

Если `max = 0` → форма не определена по стандартной шкале (уровень 0, «Иная форма»).

### Шкала форм

| Уровень | Форма |
|---:|---|
| 1 | Уполномоченное должностное лицо |
| 2 | Временный коллегиальный орган / комиссия |
| 3 | Постоянно действующий коллегиальный орган |
| 4 | Уполномоченное структурное подразделение |

### Обоснование

- Шкала отражает **возрастание организационной ёмкости** СВК — от одного лица до выделенного подразделения.
- При отметке нескольких форм берётся **максимум**: для соразмерности важна наиболее развитая из заявленных структур.
- Данные — **самоотчёт** организации в сводном отчёте.

### Пример — Алтайский политех (`form_gap = +2`)

Отмечены формы уровней 2, 3 и 4 → `svk_form_level = 4`.

## 7. Метод 4. Расчётно рекомендуемая форма

### 7.1. Полная формула

```text
base = max_direction_load_level

complex_count = |{ k : load_levelₖ ≥ L_min }|

bump = bump_by,  если complex_count ≥ N_min  и  bump включён
bump = 0,        иначе

recommended_form_level = min(max(base + bump, 0), 4)
```

Параметры по умолчанию: `L_min = 2`, `N_min = 3`, `bump_by = 1`.

### 7.2. Отклонение от рекомендации

```text
form_gap = svk_form_level − recommended_form_level
```

### Интерпретация `form_gap`

| Значение | Смысл |
|---:|---|
| ≥ +1 | Форма сильнее рекомендуемой |
| 0 | Форма соответствует рекомендации |
| −1 | Умеренный дефицит |
| −2 | Существенный дефицит |
| −3 / −4 | Критический дефицит |

### Обоснование

| Элемент модели | Почему |
|---|---|
| `base = max(load_level)` | Пиковая нагрузка по направлениям задаёт минимальную «ёмкость», которую должна обеспечивать форма СВК |
| Поправка на комплексность | Многонаправленная деятельность средней интенсивности требует более сложной координации, чем одно направление с той же пиковой нагрузкой |
| Ограничение сверху (4) | Соответствует максимальной форме в шкале отчёта |
| Термин «рекомендуемая» | Эвристика модели, **не** нормативное требование законодательства |

### Пример — Агрофизический НИИ

`load_level`: 1, 1, 2, 1 → `base = 2`, `complex_count = 1` → `bump = 0` → `recommended = 2`.  
`svk_form_level = 2` → **`form_gap = 0`**.

### Пример — Адыгейский госуниверситет

`load_level`: 2, 2, 2, 2 → `base = 2`, `complex_count = 4` → `bump = 1` → `recommended = 3`.  
`svk_form_level = 4` → **`form_gap = +1`**.

### Пример — Аграрный центр «Донской»

`load_level`: 2, 2, 4, 0 → `base = 4`, `complex_count = 3` → `bump = 1` → `recommended = min(5, 4) = 4`.  
`svk_form_level = 3` → **`form_gap = −1`**.

### Распределение `form_gap` (отчёт 2025)

| `form_gap` | Организаций |
|---:|---:|
| −4 … −1 | 236 |
| 0 | 185 |
| +1 … +3 | 264 |

## 8. Метод 5. Сравнение с организациями-аналогами (peer benchmark)

### 8.1. Формирование группы аналогов

Для организации *i* последовательно проверяются ключи группировки; выбирается **наиболее детальный** ключ, в группе которого **≥ `min_group_size`** организаций (по умолчанию 5):

```text
1) G_profile  = load_profileᵢ           — точный вектор уровней (напр. "2-2-4-0")
2) G_max_act  = max_loadᵢ + "_act" + active_directions_countᵢ  — напр. "4_act3"
3) G_max      = "max_" + max_loadᵢ      — напр. "max_4"
```

### 8.2. Формулы peer-метрик

```text
peer_form_medianᵢ = round( median{ svk_form_levelⱼ : j ∈ Gᵢ } )

form_vs_peerᵢ = svk_form_levelᵢ − peer_form_medianᵢ
```

Дополнительно сохраняются: `peer_group_key`, `peer_group_level`, `peer_group_size`.

### Интерпретация `form_vs_peer`

| Значение | Смысл |
|---:|---|
| > 0 | Форма выше типичной для аналогов |
| 0 | Форма на уровне медианы аналогов |
| < 0 | Форма ниже типичной для аналогов |

Все значения — **целые уровни** (после округления медианы).

### Обоснование

| Решение | Почему |
|---|---|
| Peer-сравнение | Отвечает на вопрос «как делают другие с похожей нагрузкой», дополняя модельную рекомендацию |
| Медиана, а не среднее | Устойчива к выбросам; при дискретной шкале 1–4 отражает «типичную» форму |
| Округление медианы до целого | Уровни формы дискретны; дробные отклонения (±0,5) неинформативны для интерпретации |
| Откат группировки | При редком профиле нагрузки точная группа может состоять из 1 организации — откат даёт репрезентативную выборку |
| `min_group_size = 5` | Компромисс между точностью сопоставления и статистической устойчивостью |

### Пример — Аграрный центр «Донской»

| Показатель | Значение |
|---|---|
| Группа | `4_act3`, n = 11 |
| `peer_form_median` | 2 |
| `svk_form_level` | 3 |
| **`form_vs_peer`** | **+1** |

При этом `form_gap = −1`: модель рекомендует форму 4, но среди аналогов медиана — 2.

### Пример — Академия комфорта

| Показатель | Значение |
|---|---|
| Группа | `4_act4`, n = 172 |
| `peer_form_median` | 3 |
| `svk_form_level` | 2 |
| **`form_vs_peer`** | **−1** |

## 9. Покрытие направлений (связанный контекст)

Покрытие **не входит** в формулы `form_gap` и `form_vs_peer`, но используется в **группах риска** и на вкладке «Направления».

### Формулы

```text
Cₖ = 1,  если направление k отмечено как покрытое СВК ("Да")
Cₖ = 0,  иначе

covered_active = Σₖ (activeₖ × Cₖ)

coverage_share_active = covered_active / active_directions_count   (если active_directions_count > 0)

uncovered_loadₖ = activeₖ × (1 − Cₖ) × load_levelₖ

uncovered_load_sum = Σₖ uncovered_loadₖ
```

### Обоснование

Соразмерность формы и полнота охвата направлений — **разные оси**. Организация может иметь адекватную форму, но не включать активное направление в контур СВК (и наоборот). Пример: Агрофизический НИИ — `form_gap = 0`, но группа риска **B** (проекты активны, но не покрыты).

## 10. Группы риска

Классификация по приоритету (первое сработавшее условие):

```text
E:  отчёт не заполнен
D:  uncovered_active_directions_count > 0  И  form_gap < 0
B:  uncovered_active_directions_count > 0
C:  form_gap < 0
A:  иначе (сбалансированная СВК)
```

### Обоснование

- Группа **C** — дефицит формы при прочих равных.
- Группа **D** — сочетание слабой формы и непокрытия; наивысший приоритет среди «содержательных» рисков.
- `form_gap = 0` **не гарантирует** группу A при непокрытых направлениях.

---

## 11. Приоритизация в топе рисков

Для ранжирования организаций (`top_risk_organizations`):

```text
risk_sort = uncovered_load_sum × 10
          + max_uncovered_load × 5
          + max(0, −form_gap) × 3
          + flag(есть замечания по качеству данных)
```

### Обоснование

Непокрытая нагрузка весит больше всего; дефицит формы (`form_gap < 0`) — дополнительный, но менее приоритетный фактор.

## 14. Ограничения и допущения

1. **Относительность нагрузки** — уровни 1–4 зависят от состава организаций в годовом отчёте.
2. **Эвристическая модель** — пороги перцентилей и правило complexity bump настраиваются в `config/scoring.yml`.
3. **Самоотчётность** — форма и покрытие из анкеты без независимой верификации.
4. **Нулевая активность** — не повышает рекомендуемую форму.
5. **Peer ≠ норматив** — медиана аналогов отражает практику в отчёте, а не эталон.
6. **Два взгляда могут расходиться** — `form_gap` и `form_vs_peer` отвечают на разные вопросы; расхождение — повод для углублённого анализа, а не автоматический признак ошибки.

## 15. Настраиваемые параметры (`config/scoring.yml`)

```yaml
load_level_thresholds:
  level_1_max_percentile: 0.75   # P₁
```
