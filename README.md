# СВК Analytics

Проект для ежегодной аналитики сводного отчета по мониторингу организации и осуществлению внутреннего контроля в подведомственных организациях.

Главная идея: каждый год вы кладете новый файл отчета в `data/raw/`, запускаете одну команду — и получаете обновленные расчеты, таблицы и интерактивный дашборд.

## Что умеет проект

- Загружает отчет из `.XLS`, `.xlsx`, `.xls`, `.html`.
- Автоматически находит нужные колонки по конфигу `config/columns.yml`.
- Считает общую статистику из аналитического обзора: статус отчетности, базовые элементы СВК, формы СВК, рискоориентированность, внутреннюю оценку эффективности, нарушения, меры реагирования и нормированные показатели.
- Считает покрытие направлений внутренним контролем:
  - финансово-хозяйственная деятельность;
  - закупки;
  - имущество;
  - проектная деятельность.
- Считает нагрузку по каждому направлению на основе фактических объемов деятельности.
- Сравнивает нагрузку с формой внутреннего контроля:
  - 1 — уполномоченное должностное лицо;
  - 2 — временный коллегиальный орган / комиссия;
  - 3 — постоянно действующий коллегиальный орган;
  - 4 — структурное подразделение.
- Выявляет организации с:
  - непокрытыми активными направлениями;
  - слабой формой СВК при высокой нагрузке;
  - двойным риском;
  - потенциальными противоречиями в данных.
- Формирует интерактивный дашборд Streamlit.
- Экспортирует расчетную таблицу и сводки в `outputs/`.

## Быстрый запуск в Cursor IDE

1. Откройте папку проекта в Cursor.
2. Положите годовой отчет в папку:

```text
data/raw/
```

Например:

```text
data/raw/Сводный отчет по СВК.XLS
```

3. Откройте встроенный терминал Cursor и выполните:

```bash
python -m venv .venv
```

4. Активируйте окружение.

Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

macOS / Linux:

```bash
source .venv/bin/activate
```

5. Установите зависимости:

```bash
pip install -r requirements.txt
```

6. Запустите проверочный расчет:

```bash
python run_analysis.py --input "data/raw/Сводный отчет по СВК.XLS" --year 2025
```

7. Запустите дашборд:

```bash
streamlit run app.py
```

В браузере откроется интерактивный дашборд.

## Ежегодный сценарий

1. Замените файл в `data/raw/` на новый отчет.
2. Запустите:

```bash
python run_analysis.py --input "data/raw/НОВЫЙ_ОТЧЕТ.XLS" --year 2026
streamlit run app.py
```

Если структура колонок изменилась, нужно поправить только `config/columns.yml`, а не переписывать код.

## Структура проекта

```text
svk_analytics_project/
├── app.py                         # Streamlit-дашборд
├── run_analysis.py                # Командный запуск расчета
├── requirements.txt               # Python-зависимости
├── config/
│   ├── columns.yml                # Карта колонок отчета
│   └── scoring.yml                # Правила расчета нагрузки и формы СВК
├── data/
│   ├── raw/                       # Сюда кладется исходный отчет
│   └── processed/                 # Сюда сохраняются обработанные данные
├── outputs/                       # Таблицы и сводки после расчета
├── docs/
│   ├── metric_passport.md         # Описание метрик
│   └── cursor_prompts.md          # Готовые промпты для Cursor
└── src/svk_analytics/
    ├── io.py                      # Загрузка файлов
    ├── columns.py                 # Поиск колонок по конфигу
    ├── cleaning.py                # Очистка значений
    ├── scoring.py                 # Расчет нагрузки и формы СВК
    ├── summaries.py               # Сводные таблицы
    └── exports.py                 # Экспорт результатов
```

## Главные выходные файлы

После `run_analysis.py` появятся. Файл `metrics_summary.xlsx` теперь содержит листы `overview`, `report_status`, `svk_elements`, `svk_form_flags`, `svk_form_level`, `risk_methodology`, `effectiveness_review`, `improvement_actions`, `violations`, `normalized_metrics`, `directions`, `risk_groups`, `top_risk_orgs`, `quality_issues`, `by_federal_district`, `by_org_type`.

```text
outputs/enriched_organizations.csv
outputs/general_statistics.csv
outputs/direction_summary.csv
outputs/risk_groups.csv
outputs/data_quality_issues.csv
outputs/metrics_summary.xlsx
```

## Как развивать проект

Лучшее место для изменений — конфиги:

- `config/columns.yml` — если в новом отчете поменялись названия колонок;
- `config/scoring.yml` — если нужно изменить правила оценки нагрузки, веса или пороги.

Код менять нужно только при добавлении принципиально новых аналитических блоков.
